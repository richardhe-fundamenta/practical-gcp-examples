5 · Concept & architecture

What happens on a request

sequenceDiagram
    actor U as User
    participant ADK as ADK proxy (Cloud Run · no LLM)
    participant MA as Managed agent (sandbox)
    participant SR as Skill Registry
    U->>ADK: "prime factors of 360?"
    ADK->>MA: stream interaction (background + stream)
    SR-->>MA: skills mounted read-only
    Note over MA: picks prime-factorizer, runs Python in the sandbox
    MA-->>ADK: SSE: reasoning → 360 = 2^3 x 3^2 x 5
    ADK-->>U: streamed straight through (A2A)

The ADK agent has no LLM — it streams the managed agent's output straight back (one LLM total · latency ≈ the managed agent alone).
