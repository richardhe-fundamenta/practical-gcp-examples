One tip before we start

Building an ADK agent, implementing A2A, and deploying to Cloud Run? Use Google's Agents CLI (google-agents-cli). It scaffolds the agent, A2A wiring, Terraform, deploy and publish flow — roughly 80% less time than wiring it all by hand. 
uv tool install google-agents-cli
agents-cli playground      # local
agents-cli deploy          # to Cloud Run
agents-cli publish gemini-enterprise --registration-type a2a ...
1 · Why

Skills are great — until they run code

  • Agents increasingly load skills: instructions + scripts, surfaced on demand.
  • The natural next step is letting them execute code — analysis, charts, transforms.
  • But that code is AI-generated / user-influenced → effectively untrusted.
  • Running it beside your app, with your creds and network, is a non-starter for enterprise.
  • Need: strong isolation · no ambient credentials · default-deny network · fast startup · disposable per run.
2 · What

The most mature setup today

  • App stays on Cloud Run — where your A2A / Gemini Enterprise integration already lives.
  • GKE is used only as a secure Agent Sandbox (Autopilot + the Agent Sandbox add-on).
  • You don't run workloads on GKE — you run untrusted code there.
  • Clean split: app plane (Cloud Run) ⇄ execution plane (GKE).
  • Bridged by an IAM-gated control-plane endpoint + an internal router.
  • Minimal infra surface; nodes stay private.

6 · Observability

Every run, logged in BigQuery

  • GenAI logging auto-exports to BigQuery → the completions_view.
  • One row per message part: who said what, every tool_call and its tool_response, tokens, finish_reasons — exactly what the model saw and produced.
  • Grouped by conversation_id; ordered by timestamp · message_idx · part_idx.
SELECT timestamp, role, part_type, tool_name, content
FROM `…a2a_with_gke_sandbox_telemetry.completions_view`
ORDER BY timestamp DESC LIMIT 10;

8 · Summary

Wrap-up & limitations

The pattern

  • App on Cloud Run.
  • Untrusted code in a GKE Agent Sandbox (gVisor, default-deny, disposable).
  • Results rendered back into Gemini Enterprise as A2UI cards.

Know the edges (mid-2026)

  • GE supports A2UI v0.8 only.
  • Streaming off — it doesn't yet render A2UI in GE; result is a task artifact.
  • So thinking is a text section just above the chart, not GE's thinking tab.
  • See the README Gotchas for the rest.
